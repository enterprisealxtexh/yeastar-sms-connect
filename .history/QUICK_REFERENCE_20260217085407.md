# Quick Reference - Production Deployment

## 🚀 One-Command Deployment

```bash
# On your Ubuntu server (as root):
wget https://your-repo-url/raw/main/deploy.sh -O deploy.sh
sudo bash deploy.sh

# Then edit configuration:
sudo nano /opt/yeastar-sms-connect/.env.production

# Restart services:
sudo systemctl restart yeastar-api yeastar-vite
```

---

## 📋 Service Commands

### Check Status
```bash
sudo systemctl status yeastar-api        # API Server
sudo systemctl status yeastar-vite       # Frontend
sudo systemctl status nginx              # Web Server
```

### View Logs
```bash
sudo journalctl -u yeastar-api -n 50     # Last 50 API logs
sudo journalctl -u yeastar-vite -n 50    # Last 50 Frontend logs
sudo journalctl -u yeastar-api -f        # Live API logs
```

### Restart Services
```bash
sudo systemctl restart yeastar-api               # Restart API
sudo systemctl restart yeastar-vite              # Restart Frontend
sudo systemctl restart yeastar-api yeastar-vite  # Restart both
sudo systemctl reload nginx                       # Reload web server
```

### Stop/Start Services
```bash
sudo systemctl stop yeastar-api    # Stop API (will stop background jobs)
sudo systemctl start yeastar-api   # Start API
sudo systemctl start yeastar-vite  # Start Frontend
```

## 🛠️ Troubleshooting

### API Server Won't Start

**Problem**: `systemctl status yeastar-api` shows failed

**Solution**:
```bash
# Check logs
sudo journalctl -u yeastar-api -n 100

# Verify database path exists
ls -la /var/lib/yeastar-sms-connect/

# Check if port is available
sudo lsof -i :2003

# Kill any orphaned process
sudo pkill -9 node

# Restart
sudo systemctl restart yeastar-api
```

### High Memory/CPU Usage

**Problem**: System seems slow

**Solution**:
```bash
# Check what's using resources
top

# Identify processes
ps aux | grep node

# Restart services
sudo systemctl restart yeastar-api yeastar-vite

# Check database integrity
sqlite3 /var/lib/yeastar-sms-connect/sms.db "PRAGMA integrity_check;"
```

### Certificate Renewal Failed

**Problem**: SSL certificate not renewing automatically

**Solution**:
```bash
# Check certbot status
sudo systemctl status certbot.timer

# Manual renewal with verbose output
sudo certbot renew --verbose

# If still fails, check renewal hook
sudo tail -f /var/log/letsencrypt/letsencrypt.log
```

### Gateway Connection Issues

**Problem**: "Failed to connect to Gateway"

**Solution**:
```bash
# Check if gateway is reachable
ping <gateway_ip>

# Test port connectivity
nc -zv <gateway_ip> 5038

# Verify credentials in .env.production
cat /opt/yeastar-sms-connect/.env.production | grep GATEWAY

# Check API logs for specific error
sudo journalctl -u yeastar-api -n 100 | grep -i gateway
```

### Database Locked

**Problem**: "database is locked" errors

**Solution**:
```bash
# Check if multiple instances running
ps aux | grep api-server

# Kill any orphaned processes
sudo pkill -9 -f api-server

# Restart API
sudo systemctl restart yeastar-api

# If persists, rebuild database connection
sudo systemctl stop yeastar-api
sleep 2
sudo systemctl start yeastar-api
```

### Site Not Loading (https://calls.alxtexh.top)

**Problem**: Connection refused or timeout

**Solution**:
```bash
# Verify Nginx is running
sudo systemctl status nginx

# Test Nginx config
sudo nginx -t

# Check if frontend is running
sudo systemctl status yeastar-vite

# Check Nginx error log
tail -f /var/log/nginx/calls.alxtexh.top_error.log

# Check backend connectivity
curl http://localhost:2003/api/health

# Restart all services
sudo systemctl restart yeastar-api yeastar-vite nginx
```

## 🎯 Performance Tuning

### Nginx Optimization (Edit: `/etc/nginx/sites-available/calls.alxtexh.top`)
```bash
# Increase worker processes
worker_processes auto;

# Increase file descriptors
worker_rlimit_nofile 200000;

# Tune buffer sizes
client_body_buffer_size 128k;
client_max_body_size 10m;

# Enable caching
proxy_cache_path /var/cache/nginx levels=1:2 keys_zone=my_cache:10m;
```

### Database Optimization
```bash
# Run maintenance
sqlite3 /var/lib/yeastar-sms-connect/sms.db "VACUUM;"
sqlite3 /var/lib/yeastar-sms-connect/sms.db "ANALYZE;"

# Check integrity
sqlite3 /var/lib/yeastar-sms-connect/sms.db "PRAGMA integrity_check;"
```

## 🆘 Emergency Procedures

### Complete System Restart
```bash
# Graceful shutdown
sudo systemctl stop yeastar-api yeastar-vite nginx

# Wait
sleep 5

# Restart all
sudo systemctl start nginx yeastar-api yeastar-vite

# Verify
sudo systemctl status yeastar-api yeastar-vite nginx
```

### Database Recovery
```bash
# Backup current (corrupted) database
cp /var/lib/yeastar-sms-connect/sms.db /var/lib/yeastar-sms-connect/sms.db.corrupted

# Restore from backup
ls -lh /var/backups/yeastar-sms-connect/
cp /var/backups/yeastar-sms-connect/sms_db_YYYYMMDD_HHMMSS.sqlite3 /var/lib/yeastar-sms-connect/sms.db

# Restart API
sudo systemctl restart yeastar-api
```

### Revert Last Code Update
```bash
cd /opt/yeastar-sms-connect

# Revert to previous version
git revert HEAD
# or
git reset --hard HEAD~1

# Rebuild
npm run build

# Restart
sudo systemctl restart yeastar-api yeastar-vite
```

## 📝 Important Notes

- **Never** edit files directly in `/opt/yeastar-sms-connect` - use git to manage code
- **Always** backup database before major changes
- **Check logs** first when troubleshooting - 90% of issues are in the logs
- **Monitor resource usage** - increase server specs if consistently >80%
- **Keep SSL certificates** renewed - set calendar reminder for 30 days before expiry
- **Review backups** regularly - test restore procedure quarterly
